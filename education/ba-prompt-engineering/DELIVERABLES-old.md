# Deliverables: Курс "Практика применения промпт-инжиниринга в работе БА"

## Обзор проекта

Данный проект содержит полный набор материалов для образовательного курса по применению промпт-инжиниринга в работе бизнес-аналитика, разработанного для команды БА Манго Телеком.

## Структура deliverables

### 1. Структура курса
**Файл:** `course-structure.md`

**Содержание:**
- Целевая аудитория и цели курса
- 6 модулей обучения с детальным описанием тем
- Рекомендуемая продолжительность каждого модуля
- Формат обучения и необходимые материалы
- Ожидаемые результаты для участников

**Общая продолжительность курса:** 3.5-4.5 часа основной части + перерывы

---

### 2. Структура презентации
**Файл:** `presentation-structure.md`

**Содержание:**
- 14 иллюстрированных слайдов
- Тезисы для каждого слайда
- Ключевые месседжи
- Дополнительные материалы и домашнее задание

**Охват слайдов:**
1. Титульный слайд
2. Зачем БА нужен промпт-инжиниринг
3. Основные возможности ИИ для БА

5. Техники промпт-инжиниринга для БА
6. Best Practices
7. Простые промпты для повседневной работы
8. Промпты для обработки транскрипций (BA_ASR)
9. Практический пример: от транскрипции к требованиям
10. Промпты для генерации и валидации ФТ (BA_FD)
11. Подготовка ИИ-модели к работе
12. Выбор ИИ-платформы
13. Интеграция в рабочий процесс БА
14. Заключение и следующие шаги

### 3. Детальное содержание презентации
**Файл:** `presentation-content.md`

**Содержание:**
- Полный текст для каждого слайда (тезисы, примеры, код)
- Детальное описание иллюстраций для каждого слайда
- Технические требования к визуализации
- Рекомендации по инструментам создания иллюстраций

**Особенности:**
- Каждое описание иллюстрации включает:
  - Концепцию визуализации
  - Детальное описание элементов
  - Цветовую схему
  - Рекомендации по стилю
- Единообразный визуальный стиль для всех слайдов
- Технические параметры (формат 16:9, разрешение, цветовая модель)

## Ключевые особенности курса

### Ссылки на промпты
Курс включает ссылки на все промпты из wiki:
- **BA_ASR_v_2_express** — быстрая очистка транскрипций
- **BA_ASR_v_6_4** — продвинутая обработка с настройками
- **BA_ASR_v_7** — оптимизированная версия для критически важных встреч
- **BA_FD_gen_v_4_2 / v_4_3** — генерация функциональных требований
- **BA_FD_valid_v3_0** — валидация функциональных требований

Все промпты доступны в: https://github.com/G-Ivan-A/BA-edu/wiki

### Практическая направленность
- Простые промпты для немедленного применения
- Реальные примеры из практики БА в ИТ-телекоме
- Пошаговый план интеграции ИИ в рабочий процесс
- Конкретные метрики эффективности (экономия до 40% времени)

### Теоретическая база
- Современные техники промпт-инжиниринга (Chain-of-Thought, Few-Shot Learning, и др.)
- Ссылка на углубленную статью: https://habr.com/ru/articles/956596/
- Соответствие международным стандартам (TM Forum Frameworx, ITIL, eTOM, SID)

### Best Practices
- Четкие рекомендации "DO" и "DON'T"
- Критерии выбора ИИ-платформы
- Roadmap внедрения от первой недели до масштабирования
- Создание командной библиотеки промптов

## Как использовать материалы

### Для преподавателя:
1. Изучите `teacher-scenario.md` — ваш основной гайд
2. Ознакомьтесь с `presentation-structure.md` для понимания логики курса
3. Используйте `presentation-content.md` как основу для создания визуальных слайдов
4. Адаптируйте примеры под специфику вашей аудитории

### Для дизайнера:
1. Откройте `presentation-content.md`
2. Для каждого слайда есть раздел "Описание иллюстрации"
3. Следуйте описанным визуальным концепциям
4. Соблюдайте технические требования (формат, разрешение, стиль)
5. Используйте рекомендованные инструменты (Figma, Canva, Adobe Illustrator)

### Для участников курса:
1. `presentation-structure.md` — обзор тем и тезисов
2. Ссылки на wiki с промптами для практической работы
3. Домашнее задание для закрепления навыков
4. Ресурсы для дальнейшего изучения

## Метрики успеха

После прохождения курса участники смогут:
1. ✅ Понимать основные принципы работы с ИИ в контексте задач БА
2. ✅ Составлять эффективные промпты для своих задач
3. ✅ Использовать готовые промпты для обработки транскрипций (BA_ASR)
4. ✅ Применять промпты для генерации и валидации ФТ (BA_FD)
5. ✅ Интегрировать ИИ-инструменты в свой рабочий процесс

**Целевые показатели:**
- Минимум 70% участников начали использовать ИИ в течение месяца
- Создано минимум 5 новых промптов командой
- Экономия времени на обработке транскрипций: 30-40%
- NPS курса > 8
