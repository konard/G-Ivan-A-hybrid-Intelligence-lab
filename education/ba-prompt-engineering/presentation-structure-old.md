### Слайд 2: Зачем БА нужен промпт-инжиниринг?
**Тезисы:**
- ИИ как инструмент повышения продуктивности БА

**Тезисы:**
- 📝 Обработка и структурирование транскрипций встреч

### Слайд 4: Что такое промпт и промпт-инжиниринг?
**Тезисы:**
- **Промпт** = задание для ИИ-модели
- **Промпт-инжиниринг** = искусство создания эффективных заданий
- Структура промпта: роль + контекст + задача + формат вывода
- Простой пример промпта для БА

**Пример:**
```
Ты — бизнес-аналитик в ИТ-телекоме.
Проанализируй следующую транскрипцию встречи
и выдели функциональные требования к системе.
Формат: нумерованный список с уровнями вложенности.
```

### Слайд 5: Техники промпт-инжиниринга для БА
**Тезисы:**
- **Chain-of-Thought (CoT):** "думай пошагово" — для сложного анализа
- **Few-Shot Learning:** примеры правильных результатов в промпте
- **Role-Playing:** назначение экспертной роли ИИ
- **Structured Output:** задание точного формата вывода
- **Iterative Refinement:** постепенное улучшение результата

**Ключевой месседж:** Правильная техника = качественный результат

---

### Слайд 6: Best Practices промпт-инжиниринга для БА
**Тезисы:**
✅ **DO:**
- Четко формулируй задачу и роль
- Предоставляй контекст и примеры
- Указывай формат вывода
- Используй терминологию предметной области
- Итеративно улучшай промпт

❌ **DON'T:**
- Не будь расплывчатым
- Не забывай про валидацию результата
- Не доверяй ИИ на 100% без проверки
- Не перегружай один промпт множеством задач

### Слайд 8: Промпты для обработки транскрипций (BA_ASR)
**Обзор семейства промптов BA_ASR:**

**BA_ASR_v_2_express** — быстрая очистка
- Удаление дубликатов, таймштампов, ошибок ASR
- Разбивка на абзацы
- Список сомнительных терминов для проверки

**BA_ASR_v_6_4** — продвинутая обработка
- Настраиваемые параметры обработки
- Автоматическое разбиение длинных текстов
- Контекстное исправление ошибок

**BA_ASR_v_7** — оптимизированная версия
- Гибкая конфигурация под разные типы встреч
- Глоссарий исправлений
- Сохранение разговорного стиля

**Ссылка:** https://github.com/G-Ivan-A/BA-edu/wiki

### Слайд 9: Практический пример: от транскрипции к требованиям
**До обработки (сырая транскрипция ASR):**
```
[00:12:34] Спикер_1: Ну вот значит э-э-э нам нужно чтобы
система э отправляла отправляла уведомления когда когда
звонок не не отвечен э-э ну типа того...
```

**После обработки промптом BA_ASR:**
```
1. Система должна отправлять уведомления,
   когда звонок остался без ответа.
```

**После извлечения требований:**
```
4.1. Система отправляет уведомление пользователю
     при пропущенном входящем звонке.
4.1.1. Уведомление отправляется не позднее 30 секунд
       после завершения звонка.
```

### Слайд 10: Промпты для генерации и валидации ФТ (BA_FD)
**Обзор семейства промптов BA_FD:**

**BA_FD_gen_v_4_2 / v_4_3** — генерация требований
- Соответствие стандартам TM Forum Frameworx
- Единая терминология телеком-домена
- Проверка на атомарность и полноту
- Формальный язык требований

**BA_FD_valid_v3_0** — валидация требований
- Проверка орфографии и пунктуации
- Устранение неоднозначностей
- Проверка на дубликаты и противоречия
- Выявление граничных случаев
- Соответствие международным стандартам (ITIL, eTOM, SID)

**Ссылка:** https://github.com/G-Ivan-A/BA-edu/wiki

### Слайд 11: Подготовка ИИ-модели к работе с промптом
**Ключевые подходы:**

**1. Контекстуализация**
- Предоставление глоссария терминов
- Описание стандартов компании
- Примеры корректных требований

**2. Few-Shot Learning**
- 2-3 примера правильно оформленных требований
- Показ желаемого формата вывода

**3. Итеративное улучшение**
- Версионирование промптов
- Анализ ошибок и корректировка
- A/B тестирование разных формулировок

**4. Chain-of-Thought**
- Пошаговое решение сложных задач
- Промежуточные рассуждения модели

**Источник:** https://habr.com/ru/articles/956596/

### Слайд 12: Выбор ИИ-платформы для БА
**Критерии выбора:**
- 🔒 Безопасность данных (On-premise vs Cloud)
- 🌍 Поддержка русского языка
- 💰 Стоимость использования
- 🔌 Возможности интеграции
- 📊 Качество работы с техническими текстами

**Популярные платформы:**
- **ChatGPT (OpenAI):** универсальность, качество
- **Claude (Anthropic):** работа с длинными документами
- **GigaChat (Сбер):** русский язык, локальное размещение
- **YandexGPT:** интеграция с Yandex Cloud
- **Локальные модели:** полный контроль, приватность

**Рекомендация:** начните с облачных решений, затем оцените необходимость on-premise
