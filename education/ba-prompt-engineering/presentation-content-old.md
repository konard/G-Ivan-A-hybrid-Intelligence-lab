## Слайд 4: Что такое промпт и промпт-инжиниринг?

### Текст слайда
**Заголовок:** Что такое промпт и промпт-инжиниринг?

**Определения:**
- **Промпт** = задание для ИИ-модели (инструкция, что и как делать)
- **Промпт-инжиниринг** = искусство создания эффективных заданий для ИИ

**Структура эффективного промпта:**
```
┌─────────────────────────────────────┐
│ 1. РОЛЬ: Кто должен "мыслить" ИИ   │
│ 2. КОНТЕКСТ: Какая предметная область│
│ 3. ЗАДАЧА: Что конкретно нужно сделать│
│ 4. ФОРМАТ: Как представить результат │
└─────────────────────────────────────┘
```

**Простой пример промпта для БА:**
```
Ты — бизнес-аналитик в ИТ-телекоме.

Проанализируй следующую транскрипцию встречи
и выдели функциональные требования к системе.

Формат: нумерованный список с уровнями
вложенности (4.1, 4.1.1, и т.д.).
```

### Описание иллюстрации
**Концепция:** Визуальная схема структуры промпта с примером

**Элементы визуализации:**
1. **Левая часть:** Четыре блока-компонента промпта:
   - **Блок "РОЛЬ"** (синий): Иконка бейджа с галочкой
   - **Блок "КОНТЕКСТ"** (зеленый): Иконка книги/базы знаний
   - **Блок "ЗАДАЧА"** (оранжевый): Иконка чек-листа
   - **Блок "ФОРМАТ"** (фиолетовый): Иконка документа с структурой

2. **Стрелки:** Показывают последовательность компонентов сверху вниз

3. **Правая часть:** Визуализация примера:
   - Текстовое поле с выделенными цветом частями:
     - "Ты — бизнес-аналитик..." (синий фон)
     - "в ИТ-телекоме" (зеленый фон)
     - "Проанализируй... выдели требования" (оранжевый фон)
     - "Формат: нумерованный список..." (фиолетовый фон)

4. **Центральная иконка:** ИИ-модель (стилизованный мозг/чип) обрабатывает промпт

5. **Внизу:** Стрелка к результату (структурированный список требований)

**Стиль:** Схематичный, образовательный, с четким разделением цветом

## Слайд 5: Техники промпт-инжиниринга для БА

### Текст слайда
**Заголовок:** Техники промпт-инжиниринга для БА

**Основные техники:**

1. **Chain-of-Thought (CoT)** — Цепочка рассуждений
   - Добавь: "Думай пошагово" или "Объясни свой ход мыслей"
   - Используй для: сложного анализа требований

2. **Few-Shot Learning** — Обучение на примерах
   - Добавь: 2-3 примера правильных результатов в промпт
   - Используй для: генерации требований по стандарту

3. **Role-Playing** — Назначение роли
   - Добавь: "Ты — эксперт в телеком-домене..."
   - Используй для: получения специализированных ответов

4. **Structured Output** — Структурированный вывод
   - Добавь: точное описание формата (JSON, таблица, список)
   - Используй для: интеграции с другими системами

5. **Iterative Refinement** — Итеративное улучшение
   - Добавь: "Сначала предложи варианты, затем улучши лучший"
   - Используй для: критически важных документов

**Ключевой месседж:**
> Правильная техника = качественный результат

### Описание иллюстрации
**Концепция:** Пять карточек с техниками, каждая с иконкой и визуальным примером

**Элементы визуализации:**
1. **Карточка 1 (CoT):**
   - Иконка: Цепь из звеньев
   - Мини-схема: Проблема → Шаг 1 → Шаг 2 → Шаг 3 → Решение
   - Цвет: Синий

2. **Карточка 2 (Few-Shot):**
   - Иконка: Три звезды (примеры)
   - Мини-схема: Пример 1 ✓ + Пример 2 ✓ + Пример 3 ✓ → Новый результат
   - Цвет: Зеленый

3. **Карточка 3 (Role-Playing):**
   - Иконка: Театральная маска / бейдж эксперта
   - Мини-схема: Шляпа эксперта на ИИ-модели
   - Цвет: Оранжевый

4. **Карточка 4 (Structured Output):**
   - Иконка: Таблица/сетка
   - Мини-схема: Хаотичные данные → Структурированная таблица
   - Цвет: Фиолетовый

5. **Карточка 5 (Iterative Refinement):**
   - Иконка: Циклическая стрелка
   - Мини-схема: Версия 1 → Версия 2 → Версия 3 (улучшение)
   - Цвет: Красный

**Расположение:** Пять карточек веером или в виде пентагона

**Стиль:** Яркие цвета, четкие иконки, современный flat design

## Слайд 6: Best Practices промпт-инжиниринга для БА

### Текст слайда
**Заголовок:** Best Practices промпт-инжиниринга для БА

**✅ DO — Делай так:**
- ✓ Четко формулируй задачу и роль ИИ
- ✓ Предоставляй достаточный контекст и примеры
- ✓ Указывай точный формат вывода
- ✓ Используй терминологию предметной области
- ✓ Итеративно улучшай промпт на основе результатов
- ✓ Сохраняй успешные промпты в библиотеку команды

**❌ DON'T — Не делай так:**
- ✗ Не будь расплывчатым в формулировках
- ✗ Не забывай про валидацию результата человеком
- ✗ Не доверяй ИИ на 100% без проверки
- ✗ Не перегружай один промпт множеством разных задач
- ✗ Не игнорируй специфику своего домена
- ✗ Не используй промпт "как есть" без адаптации

### Описание иллюстрации
**Концепция:** Две колонки с контрастными визуальными примерами

**Элементы визуализации:**
1. **Левая колонка (✅ DO):**
   - Цвет фона: Светло-зеленый
   - Иконка: Большая зеленая галочка сверху
   - Шесть мини-иллюстраций:
     1. Целевая мишень (четкая цель)
     2. Контекстное облако с примерами
     3. Шаблон документа
     4. Глоссарий терминов
     5. Циклическая стрелка улучшения
     6. Папка с сохраненными промптами

2. **Правая колонка (❌ DON'T):**
   - Цвет фона: Светло-красный
   - Иконка: Большой красный крестик сверху
   - Шесть мини-иллюстраций с перечеркиванием:
     1. Облако с вопросительными знаками (неясность)
     2. Робот без надзора человека
     3. Знак "100% доверие" перечеркнут
     4. Перегруженный промпт (много стрелок в разные стороны)
     5. Игнорирование специфики (общий шаблон для всего)
     6. Копирование без изменений (Ctrl+C→Ctrl+V зачеркнуто)

3. **Центральная разделительная линия:** Четкая граница между "правильно" и "неправильно"

4. **Визуальный стиль:**
   - Иконки в стиле line-art с заливкой
   - Читаемые, простые образы
   - Контрастные цвета для быстрого восприятия

**Формат:** Симметричное разделение слайда 50/50

## Слайд 8: Промпты для обработки транскрипций (BA_ASR)

### Текст слайда
**Заголовок:** Промпты для обработки транскрипций встреч (BA_ASR)

**Семейство промптов BA_ASR:**

**🚀 BA_ASR_v_2_express** — Быстрая очистка
- ✓ Удаление дубликатов, таймштампов, ошибок ASR
- ✓ Разбивка на логические абзацы
- ✓ Список сомнительных терминов для проверки
- **Когда использовать:** Быстрая обработка коротких встреч

**⚙️ BA_ASR_v_6_4** — Продвинутая обработка
- ✓ Настраиваемые параметры обработки (5 режимов)
- ✓ Автоматическое разбиение длинных текстов на блоки
- ✓ Контекстное исправление ошибок с глоссарием
- **Когда использовать:** Длинные встречи, нужна точная настройка

**🎯 BA_ASR_v_7** — Оптимизированная версия
- ✓ Гибкая конфигурация под разные типы встреч
- ✓ Детальный глоссарий принятых исправлений
- ✓ Сохранение разговорного стиля и интонаций
- **Когда использовать:** Критически важные встречи с клиентами

**📚 Ссылка на wiki:** https://github.com/G-Ivan-A/BA-edu/wiki

### Описание иллюстрации
**Концепция:** Три уровня обработки транскрипций (базовый, продвинутый, экспертный)

**Элементы визуализации:**
1. **Левая часть:** Входные данные
   - Иконка микрофона
   - Аудио-волна
   - Стрелка вниз
   - Текст с ошибками ASR (красные подчеркивания, дубликаты, метки времени)

2. **Центральная часть:** Три пути обработки (вертикальные дорожки)

   **Дорожка 1 (BA_ASR_v_2_express):**
   - Цвет: Зеленый
   - Иконка: Ракета (скорость)
   - Визуализация процесса: Простая одношаговая очистка
   - Время: ~2 минуты

   **Дорожка 2 (BA_ASR_v_6_4):**
   - Цвет: Оранжевый
   - Иконка: Шестеренки (настройка)
   - Визуализация процесса: Многоэтапная обработка с настройками
   - Время: ~5 минут

   **Дорожка 3 (BA_ASR_v_7):**
   - Цвет: Фиолетовый
   - Иконка: Мишень (точность)
   - Визуализация процесса: Глубокая обработка с контекстным анализом
   - Время: ~7 минут

3. **Правая часть:** Результаты
   - Чистый текст без ошибок
   - Структурированные абзацы
   - Глоссарий исправлений (для v_6_4 и v_7)

4. **Внизу:** Выбор промпта в зависимости от критериев:
   - Скорость → v_2_express
   - Гибкость → v_6_4
   - Точность → v_7

**Стиль:** Схема технологического процесса (workflow diagram)

## Слайд 9: Практический пример: от транскрипции к требованиям

### Текст слайда
**Заголовок:** Практический пример: от транскрипции к требованиям

**Этап 1: Сырая транскрипция ASR**
```
[00:12:34] Спикер_1: Ну вот значит э-э-э нам нужно
чтобы система э отправляла отправляла уведомления
когда когда звонок не не отвечен э-э ну типа того...
[00:12:58] Спикер_2: Ага ага да да понял
```

**↓ Применяем BA_ASR_v_7**

**Этап 2: Очищенная транскрипция**
```
Спикер 1: Нам нужно, чтобы система отправляла
уведомления, когда звонок остался без ответа.

Спикер 2: Понял.
```

# Технические требования к иллюстрациям

## Общие параметры
- **Формат слайдов:** 16:9 (1920×1080 px)
- **Разрешение:** Не менее 150 DPI для печати
- **Формат файлов:** PNG (для иллюстраций с прозрачностью), JPG (для фотографий)
- **Цветовая модель:** RGB

## Стилистические требования
- **Единый стиль:** Все иллюстрации в едином визуальном стиле (flat design, line art, или modern infographic)
- **Цветовая палитра:**
  - Основные: Фирменные цвета Манго Телеком
  - Дополнительные: Синий (#2196F3), Зеленый (#4CAF50), Оранжевый (#FF9800), Фиолетовый (#9C27B0), Красный (#F44336)
- **Шрифты:**
  - Заголовки: Sans-serif, жирный (Open Sans Bold, Roboto Bold)
  - Текст: Sans-serif, обычный (Open Sans Regular, Roboto Regular)
  - Код: Monospace (Fira Code, Consolas)
- **Читаемость:** Все элементы должны быть хорошо видны при проецировании на экран
- **Доступность:** Контрастность не менее 4.5:1 для текста

## Инструменты создания
Рекомендуемые инструменты для создания иллюстраций:
- **Figma** — для инфографики и схем
- **Canva** — для быстрого создания слайдов
- **Adobe Illustrator** — для профессиональной векторной графики
- **PowerPoint/Google Slides** — встроенные инструменты SmartArt
- **ИИ-генераторы** (опционально):
  - DALL-E, Midjourney — для создания концептуальных иллюстраций
  - Stable Diffusion — для кастомизированной генерации

## Примечания к реализации
- Все иллюстрации должны поддерживать основную идею слайда
- Избегайте перегруженности: максимум 5-7 основных элементов на иллюстрации
- Используйте визуальную иерархию: главное — крупнее и ярче
- Добавляйте небольшие пояснительные подписи к сложным элементам
- Все иконки должны быть из одного набора для единообразия
