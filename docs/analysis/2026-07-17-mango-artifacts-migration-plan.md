# План миграции артефактов Mango при разделении репозиториев (B-080)

> Режим: **Analysis (recommendation)** для issue
> [#436](https://github.com/G-Ivan-A/hybrid-Intelligence-lab/issues/436) и
> backlog [B-080](../../pr-ops/backlog.md).
> Граница: **этот документ не создаёт репозитории (B-081/B-082), не переносит
> файлы (B-083) и не настраивает синхронизацию (B-084).** Он фиксирует
> исполнимый контракт миграции как upstream-вход для этих задач.

## 1. Summary / BLUF

План миграции построен на **фактическом снимке** `G-Ivan-A/mango_ba_prompts`
(commit `295b65d`, снят 2026-07-17), а не на концептуальном списке каталогов:
репозиторий содержит **5 346 файлов**, из них **1 337 файлов упоминают бренд
«Mango»**. Маппинг ниже покрывает все 11 фактических корневых каталогов и 13
корневых файлов.

Четыре вывода определяют исполнимость плана:

1. **Fail-safe работает: 98,6% объёма уходит в приватное репо.** Публичными
   кандидатами являются только `patterns/` (15 файлов), часть `standards/`
   (15 файлов) и часть `docs/adr/` — порядка **60 файлов из 5 346**. Всё
   остальное (`kb/` — 5 077 файлов, `runs/` — 78, `prompts/` — 33) приватно по
   умолчанию.
2. ⚠️ **Блокирующее расхождение с ADR-009: репозиторий `mango_ba_prompts`
   сейчас ПУБЛИЧНЫЙ** (`"visibility": "public"`, проверено через GitHub API
   2026-07-17), тогда как ADR-009 в разделе Context исходит из того, что он
   «вынужденно остаётся закрытым». Это меняет модель угрозы: разделение — не
   превентивная мера против утечки, а **реакция на уже состоявшуюся
   публикацию**. Вынесено как Open Question **Q1** — требует решения фаундера
   **до** B-083.
3. ⚠️ **Целевая структура ADR-009 не покрывает 95% объёма Mango.** ADR-009
   отводит приватному репо каталог `kb/processes/`, но фактически в Mango есть
   `kb/processed/` (5 041 файл, **138 МБ**, PDF через Git LFS), `kb/sources/`
   (31 файл), `kb/fragments/` и `kb/practices/`. Каталога `kb/processes/` не
   существует. Для `kb/sources/` и `kb/processed/` в ADR-009 **нет целевого
   дома**. Вынесено как Open Question **Q2**.
4. **Четыре целевых каталога ADR-009 — greenfield, а не миграция.** В Mango
   физически отсутствуют `evals/`, `internal-rfc/`, `internal-docs/`
   (приватные цели) и `templates/`, `app/`, `prompt-library/` (публичные цели).
   Запрет `education/` в публичном репо — **превентивный**: каталога
   `education/` в Mango нет, мигрировать нечего.

Обезличивание — не косметика: **14 из 15** файлов `standards/` и **9 из 15**
файлов `patterns/` содержат бренд, а `standards/product-classification-contract.md`
содержит 29 упоминаний и мигрируем только через переписывание. Отдельный класс
риска — `standards/team-directory.md` (реестр реальных команд, персональные
данные) и ~200 внутренних GitHub-ссылок на `mango_ba_prompts`, которые в
публичном репо станут одновременно битыми и брендирующими.

Рекомендуемое движение: **B-080 → review**; B-083 не стартует до закрытия Q1 и
Q2 фаундером.

## 2. Context / Scope

### 2.1. Причина

[ADR-009](../adr/2026-07-adr-009-mango-repo-split.md) зафиксировал решение
фаундера от 2026-07-10 разделить `mango_ba_prompts` на публичный
`ai-ba-playbooks` (архетип B по [ADR-001](../adr/2026-06-adr-001-ecosystem-infrastructure-methodology.md))
и приватный `mango-ba-prompt-library`. ADR задаёт **целевые каталоги и нормы**,
но сознательно делегирует детальный маппинг в B-080. Без этого маппинга B-081,
B-082 и B-083 не имеют исполнимого контракта: неизвестно, что именно создавать и
что куда переносить.

## 4. Правила классификации

Правила применяются **в указанном порядке**; первое сработавшее определяет
целевой репозиторий.

| № | Правило | Следствие |
| --- | --- | --- |
| R1 | **Fail-safe по умолчанию.** Если стратегия артефакта неочевидна или спорна — артефакт **приватный**. | Сомнение никогда не разрешается в пользу публикации. |
| R2 | **Операционка → приватный.** Прогоны, боевые промпты, golden-sets, внутренние RFC/аудиты, БЗ на реальных данных заказчика. | ADR-009 §2. |
| R3 | **Приватные данные и PII → приватный, без исключений.** Реальные имена, команды, контакты, внутренние URL, данные клиентов. | Не подлежит обезличиванию — подлежит невключению. |
| R4 | **Методология без привязки к бренду → публичный** после обезличивания. | ADR-009 §1. |
| R5 | **Бренд «Mango» в публичном репо запрещён** — в именах, содержимом, ссылках, скриншотах, метаданных и путях. | ADR-009 §1. |
| R6 | **`education/` в публичном репо отсутствует категорически.** | ADR-009 §1 + [ADR-007](../adr/2026-07-adr-007-hub-root-structure.md). |
| R7 | **Стоимость обезличивания > ценности артефакта → приватный.** Если файл переписывается «с нуля», это не миграция, а новая работа: её место — backlog, а не B-083. | Anti-Inflation. |
| R8 | **Методология Хаба не форкается.** Публичный репо импортирует её из Хаба, а не копирует. | ADR-009 §4. |

### 4.1. Процесс обезличивания (для R4)

Обязателен для **каждого** файла, идущего в `ai-ba-playbooks`. Выполняется в
B-083 пофайлово, результат проверяется человеком:

| Шаг | Действие | Проверка |
| --- | --- | --- |
| A1 | Заменить бренд: `Mango`, `Mango Office`, `mango_ba_prompts`, `mangosip.ru`, `МАНГО` → generic (`telecom provider`, `оператор связи`) | `grep -ri "mango"` → 0 совпадений |
| A2 | Заменить названия внутренних систем и продуктов (`МТалкер`/`mtalker`, `ВАТС`/`vpbx`, `ЛК`, `mDialogi`, `Wallboard`, `Speech Analytics`) на generic telecom-эквиваленты | ручное ревью списком из A2 |
| A3 | Удалить внутренние URL: ссылки на `github.com/G-Ivan-A/mango_ba_prompts` (~200 вхождений), issue/PR-ссылки, `g-ivan-a.github.io/mango_ba_prompts` | `grep -rE "mango_ba_prompts\|mangosip"` → 0 |
| A4 | Удалить PII: реальные имена, коды команд, контакты, e-mail (включая примеры вида `ivanov@mangosip.ru`) | ручное ревью |
| A5 | Заменить примеры на generic telecom-кейсы; сохранить методологическую суть, а не фактуру заказчика | ручное ревью |
| A6 | Обнулить `related_issues`/`related_artifacts` во frontmatter, указывающие на приватный периметр | `grep` по frontmatter |
| A7 | Проверить бинарные файлы отдельно: скриншоты и изображения **не обезличиваются грепом** | ручной просмотр каждого PNG |
| A8 | Финальный gate: полнотекстовый скан всего публичного дерева **перед первым push** | `grep -ril "mango\|mangosip\|vpbx\|mtalker"` → 0 |

⚠️ **A8 обязателен именно до первого push:** после публикации откат невозможен —
git-история и форки сохраняют содержимое (см. Q1).

## 6. Пограничные кейсы

### 6.1. `standards/GLOSSARY.md` — единственный настоящий edge case

Глоссарий одновременно содержит generic BA-термины (переиспользуемы) и термины
продуктов Mango (приватны). Три стратегии из issue #436:

| Стратегия | Плюсы | Минусы | Вердикт |
| --- | --- | --- | --- |
| Копировать в оба репо целиком | Дёшево, один шаг | ⛔ Нарушает R3/R5: приватные термины попадают в публичный репо. Плюс дрейф двух копий | **Отклонено** |
| SSOT в приватном + ссылка из публичного | Нет дрейфа, один источник | ⛔ Ссылка на приватный репо из публичного = битая ссылка + раскрытие внутренней структуры | **Отклонено** |
| Выделить shared-модуль (submodule) | Формально без дублирования | ⛔ ADR-009 прямо отклонил submodules как сохраняющие риск утечки при ошибке конфигурации. Инфраструктура дороже одного файла | **Отклонено** |
| **Расщепление по признаку приватности** | Соответствует R1/R3/R5; каждый репо самодостаточен; граница проверяема грепом | Требует ручного ревью каждого термина; допускает дрейф generic-части | ✅ **Выбрано** |

**Обоснование выбора.** Три отклонённых варианта исходят из посылки, что
глоссарий — атомарный артефакт, который нужно целиком куда-то поместить. Но
глоссарий — **список независимых терминов**, и граница приватности проходит
**внутри** него, а не по нему. Расщепление — единственная стратегия, которая
режет по фактической границе, а не вопреки ей. Дрейф generic-части приемлем:
термины BA стабильнее операционки, а ADR-009 §4 уже задал канал возврата
зрелых определений в Хаб через RFC.

**Правило исполнения (B-083):** термин уходит в публичный глоссарий, только если
он остаётся осмысленным **без** знания продуктов Mango. Иначе — приватный (R1).

## 8. Фазы миграции

Последовательность выбрана по принципу **«сначала безопасное, необратимое —
последним»**: публичный push — единственный необратимый шаг, поэтому он идёт
после всех проверок.

| Фаза | Содержание | Задача | Выход / Gate |
| --- | --- | --- | --- |
| **Фаза 0. Решения фаундера** | Закрыть Q1 (текущая публичность) и Q2 (дом для `kb/`). Без них маппинг неисполним | B-080 (эта фаза) | ✅ Ответы на Q1, Q2 зафиксированы |
| **Фаза 1. Аудит и заморозка** | Обновить снимок Mango; переприменить §5; заморозить изменения в Mango на время миграции; финальный скан на приватные данные | B-083 (подготовка) | ✅ Снимок актуален, freeze объявлен |
| **Фаза 2. Приватный репо** | Создать `mango-ba-prompt-library` (структура ADR-009, **без GitHub Actions**); перенести операционку: `prompts/`, `kb/`, `runs/`, `governance/`, `scripts/`, внутренние `docs/`, root-контракты | B-082 → B-083 | ✅ Числа сходятся (5 041 файл `kb/`, 78 `runs/`, 33 `prompts/`); локальные валидаторы зелёные |
| **Фаза 3. Обезличивание** | Прогнать A1–A7 по публичным кандидатам (`patterns/`, часть `standards/`, часть `docs/adr/`, `docs/ba-processes/`, `taxonomy.md`) **в приватной ветке/периметре** | B-083 | ✅ Ручное ревью человеком пройдено |
| **Фаза 4. Публичный репо** | Создать `ai-ba-playbooks` (архетип B, чистая история, свои `README`/`LICENSE`/CI, `templates/`, `app/`, пустой `prompt-library/telecom/`); залить только прошедшее Фазу 3 | B-081 → B-083 | ⚠️ **Gate A8: `grep -ril "mango\|mangosip\|vpbx\|mtalker"` → 0 ДО первого push**; подтвердить отсутствие `education/` |

**Почему Фаза 2 раньше Фазы 4:** приватный перенос обратим (репо закрыт, ошибка
исправляется коммитом), публичный — нет. Обезличивание (Фаза 3) выполняется
**внутри приватного периметра**, чтобы промежуточные, ещё не обезличенные
состояния никогда не попали в публичный git.

## 9. Open Questions (требуют решения фаундера)

### Q1. ⚠️ `mango_ba_prompts` уже публичный — что это меняет?

**Факт:** GitHub API на 2026-07-17 возвращает `"visibility": "public"`,
`"private": false`, `has_pages: true` (Pages опубликован). ADR-009 в разделе
Context утверждает: «Приватные данные не могут быть опубликованы без риска
утечки, поэтому репозиторий вынужденно остаётся закрытым».

**Почему это блокирует:** весь план исходит из того, что разделение
**предотвращает** утечку. Если репозиторий публичен, то 5 077 файлов `kb/`, 78
прогонов и 33 боевых промпта **уже опубликованы**, и B-083 не предотвращает
утечку, а разбирает её последствия. Это меняет и приоритет фаз, и смысл Gate A8.

| Вариант | Действие | Цена |
| --- | --- | --- |
| **V1 (рекомендуется)** | Немедленно перевести `mango_ba_prompts` в private **до** B-083; провести отдельную оценку последствий публикации (что и как долго было доступно, есть ли форки); зафиксировать результат как Audit | Останавливает продолжающееся раскрытие сразу; блокирует Pages (`has_pages: true`) — потребуется решение по витрине |
| V2 | Оставить публичным до завершения миграции, затем закрыть | Раскрытие продолжается всё время миграции; риск растёт с каждым днём |
| V3 | Признать содержимое фактически публичным и пересмотреть предпосылку ADR-009 | Требует нового ADR; противоречит R3 (в `kb/` — материалы продуктов, в `standards/team-directory.md` — PII) |

**Рекомендация исполнителя: V1**, по R1 (fail-safe). Утечка, которая уже
произошла, — не основание разрешить ей продолжаться. Отдельно: возможно,
расхождение объясняется тем, что Context ADR-009 описывает *целевое* состояние,
а не фактическое — но проверить это может только фаундер.

### Q2. ⚠️ Где дом для `kb/sources/` и `kb/processed/`?

**Факт:** ADR-009 отводит приватному репо `kb/processes/`. Фактически в Mango
есть `kb/processed/` (5 041 файл, 138 МБ), `kb/sources/` (31), `kb/fragments/`
(1), `kb/practices/` (1). Каталога `kb/processes/` не существует — вероятно,
опечатка либо сознательно более узкий scope. **95% объёма репозитория не имеет
целевого пути.**

| Вариант | Действие | Цена |
| --- | --- | --- |
| **V1 (рекомендуется)** | Прочитать `kb/processes/` в ADR-009 как опечатку и перенести `kb/` целиком, сохранив внутреннюю структуру (`sources/`, `processed/`, `fragments/`, `practices/`) | Не требует нового ADR (уточнение факта, а не пересмотр решения); сохраняет работающий пайплайн `scripts/kb/` |
| V2 | Перенести только `kb/fragments/` + `kb/practices/` в `kb/processes/`, а `sources/`+`processed/` архивировать вне репо | Разрывает `scripts/kb/`; 138 МБ БЗ теряют дом и версионирование |
| V3 | Уточнить ADR-009 отдельным decision record | Формально чисто, но блокирует B-082/B-083 на цикл ADR |

**Рекомендация исполнителя: V1.** Это единственный вариант, при котором
операционка остаётся работоспособной. В любом случае `kb/` **приватен** — вопрос
только о целевом пути внутри приватного репо, поэтому риска утечки решение не
несёт.

### Q3. Судьба `site/` и публичной витрины

`site/` (11 файлов, 9 с брендом) — витрина боевых промптов на GitHub Pages.
ADR-009 предусматривает в Pub опциональный `app/`.

| Вариант | Действие |
| --- | --- |
| **V1 (рекомендуется)** | `site/` → Priv as-is (R1); публичный `app/` — greenfield в B-081 поверх обезличенного `prompt-library/`. Витрина боевых промптов и витрина методологии — разные продукты |
| V2 | Обезличить `site/` и перенести в Pub `app/` | `generate-pages-data.mjs` генерирует данные из боевых промптов — риск утечки через сгенерированный `site/data` |
| V3 | Отказаться от публичной витрины на старте | Дёшево; ADR-009 помечает `app/` как опциональный |

## 10. Recommendations

1. **B-080 → `review`.** План исполним для B-081/B-082 в части структуры; для
   B-083 — после закрытия Q1 и Q2.
2. **Q1 закрыть до всех остальных задач.** Пока `mango_ba_prompts` публичен, план
   решает не ту задачу, которую описывает ADR-009. Рекомендация — V1 (закрыть
   репозиторий, оценить последствия отдельным Audit).
3. **Q2 закрыть до B-082.** Без целевого пути для `kb/` (95% объёма) структура
   приватного репо не определена.
4. **Не начинать B-083 до Фазы 0.** Фазы 1–5 исполнимы только с ответами на Q1/Q2.
5. **Gate A8 сделать обязательным условием первого push в публичный репо** и
   зафиксировать его в DoD задачи B-081, а не только в этом плане.
6. **Обновить снимок в Фазе 1.** Маппинг §5 привязан к `295b65d`; репозиторий
   активен.
7. **Считать `prompt-library/telecom/`, `evals/`, `templates/`, `app/`
   greenfield-работой**, а не миграцией: их наполнение не входит в B-083 и
   требует отдельных задач в backlog.

Решений этот документ не принимает: Q1–Q4 остаются за фаундером, а нормы —
за ADR-009.
