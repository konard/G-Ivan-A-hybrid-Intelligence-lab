# Ретроспектива AI-collaboration: сессия проектирования миграции Mango (Хаб → спок)

## Введение

Этот артефакт фиксирует ошибки AI-агента, выявленные в сессии по проектированию
стратегии миграции проекта Mango из Хаба (`hybrid-Intelligence-lab`) в отдельный
spoke-репозиторий. Цель документа — сохранить факты ретроспективы в
`research/hub/` как исследовательский артефакт и сформулировать системные выводы
для будущего усиления правил работы AI, **не** изменяя governance-контракты в
рамках этой задачи (см. [Ограничения](#ограничения)).

Операционный режим: `Research` — анализ фактов и извлечение системных уроков.
Источники для анализа: история диалога сессии проектирования миграции Mango,
[`AI_GOVERNANCE.md`](../../ai-governance/ai-governance.md),
[`standards/issue-workflow.md`](../../standards/issue-workflow.md),
[`pr-ops/repo-model.md`](../../pr-ops/repo-model.md),
[`ai-rules/agent-onboarding-protocol.md`](../../ai-rules/agent-onboarding-protocol.md) и
[`standards/file-naming.md`](../../standards/file-naming.md).

> **Scope:** `repo-wide`. Документ дополняет, а не заменяет ретроспективу сессии
> проектирования шаблонов споков
> ([`2026-06-02-ai-collaboration-retrospective.md`](2026-06-02-ai-collaboration-retrospective.md)):
> здесь разбирается отдельная сессия (миграция Mango), включая
> архитектурные ошибки размещения артефактов, которых в той сессии не было.

## Системные выводы

Выводы предназначены для будущего proposal или human review правил
[`AI_GOVERNANCE.md`](../../ai-governance/ai-governance.md) и
[`ai-rules/agent-onboarding-protocol.md`](../../ai-rules/agent-onboarding-protocol.md); в рамках
этой задачи governance-файлы не меняются.

1. **Operating mode — проверяемое значение из закрытого списка.** AI-агент
   выбирает только режимы, явно перечисленные в `AI_GOVERNANCE.md`
   (`Structured`, `Research`, `Education`, `Project`). Комбинированные ярлыки вроде `Research + Structured` не
   допускаются: при смешанной задаче описывается доминирующий режим и его
   ограничения.

2. **Pre-flight чтение контекста — обязательный gate, а не рекомендация.**
   Перед генерацией структуры, файлов, плана миграции или DoD агент проходит
   предполётный протокол `agent-onboarding-protocol.md`: issue → последние комментарии →
   `AI_GOVERNANCE.md` → ближайший workflow/standard → текущий PR context →
   *Readback* → стоп до апрува человека.

3. **Anti-Inflation проверяется до предложения новой структуры.** Любой новый
   каталог, файл или стандарт обязан отвечать на вопрос: какую повторяющуюся
   путаницу, review pain, ownership gap или unverifiable work он снижает. Если
   операционной боли пока нет — фиксируется `Триггер внедрения`, а артефакт
   откладывается, а не создаётся «для полноты дерева».

4. **Размещение артефакта и формат issue — часть traceability.** Тип и scope
   артефакта определяются по `repo-model.md` (term-контракты → `standards/`,
   эксперименты проекта → `projects/<spoke>/experiments/`, repo-wide
   исследования → `research/hub/`). Поля `User Story / ФТ`, operating mode, DoD
   и формат коммита нужны, чтобы связать задачу, PR и итоговый артефакт, и не
   сокращаются как «лишние» без явного решения человека.

## Ограничения

- Ретроспектива фиксирует только ошибки, перечисленные в issue #124, и не
  добавляет новые предполагаемые ошибки (режим `Research`: оперируем фактами
  диалога и текстом стандартов).
- Документ не изменяет [`AI_GOVERNANCE.md`](../../ai-governance/ai-governance.md) или другие
  governance-контракты; системные выводы предназначены для будущего proposal или
  human review.
- Термины «ретроспектива» и «корневая причина», использованные в этом артефакте,
  добавлены в [`standards/glossary.md`](../../standards/glossary.md) как единый
  источник определений.
