# Ретроспектива AI-collaboration по сессии проектирования шаблонов споков

## Введение

Этот артефакт фиксирует ошибки AI-агента, выявленные в сессии проектирования
шаблонов spoke-репозиториев и постановки задач для Хаба. Цель документа -
сохранить факты ретроспективы в `research/hub/` и сформулировать системные
выводы для будущего усиления правил работы AI, не меняя governance-файлы в
рамках этой задачи.

Операционный режим: `Research`. Источники для анализа: issue #95,
`AI_GOVERNANCE.md`, `standards/issue-workflow.md`,
`governance/repo-model.md`.

## Системные выводы

1. **Operating mode должен быть проверяемым значением.** AI-агент выбирает
   только режимы, явно перечисленные в `AI_GOVERNANCE.md`. Если задача выглядит
   смешанной, агент описывает доминирующий режим и ограничения, но не вводит
   новый ярлык.

2. **Pre-flight чтение контекста должно быть обязательным шагом.** Перед
   генерацией структуры, файлов или DoD AI должен прочитать issue, последние
   комментарии, `AI_GOVERNANCE.md`, ближайший workflow/standard и текущий PR
   context.

3. **Anti-Inflation проверяется до предложения новой структуры.** Любой новый
   файл, каталог или стандарт должен отвечать на вопрос: какую повторяющуюся
   путаницу, review pain, ownership gap или unverifiable work он снижает.

4. **Формат issue является частью traceability, а не оформлением.** Поля
   `User Story / ФТ`, operating mode, артефакты, DoD и формат коммита нужны,
   чтобы связать задачу, PR и итоговый артефакт. AI не должен сокращать их как
   "лишние" без явного решения человека.

## Ограничения

Ретроспектива фиксирует только ошибки, перечисленные в issue #95, и не
добавляет новые предполагаемые ошибки. Документ не изменяет `AI_GOVERNANCE.md`
или другие governance-контракты; выводы предназначены для будущего proposal или
human review.
