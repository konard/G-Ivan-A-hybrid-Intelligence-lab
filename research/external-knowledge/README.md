# Research: External Knowledge

Направление фиксирует **систему интеграции внешних знаний** Хаба: единый реестр
внешних источников и механизм хранения извлечённых инсайтов с привязкой к
жизненному циклу знаний. Цель — превратить ad-hoc исследование внешних статей,
репозиториев и международных практик в воспроизводимый поток
`внешний источник → инсайт → валидация → практика`, общий для всей экосистемы.

`scope: repo-wide` — механизм применяется ко всему репозиторию и транслируется в
проекты экосистемы через Base Registry (Хаб) + Local Extension (проекты).

> 🧭 **Назначение в одну строку.** Внешние источники — это
> Observation/Research, а не практика: они живут в `research/`, пока не пройдут
> валидацию и не будут переведены в `practices/`/`standards/` по
> [governance/rfc/knowledge-lifecycle-proposal.md](../../governance/rfc/knowledge-lifecycle-proposal.md).

## Правила работы с внешними источниками

Эти правила определяют, **как распознать тип задачи** про внешнее знание, **куда
фиксировать результат** и **что считать чистотой** записей. Цель — превратить
изучение источника в предсказуемый поток и не раздувать базу знаний мусором
(Anti-Inflation).

### Распознавание задачи

| Сигнал в задаче | Тип работы | Действие агента |
| --- | --- | --- |
| Метка `research` / «исследование» без конкретной ссылки | **Широкий поиск** | Ищем несколько источников по теме, каждый оцениваем отдельно. |
| Дан конкретный URL (статья, репозиторий, стандарт) | **Изучение конкретного источника** | Изучаем именно этот источник, не расширяя scope без запроса. |

### Фиксация результата

| Исход изучения | Куда писать | Куда НЕ писать |
| --- | --- | --- |
| Источник изучен (всегда, независимо от текущей пользы) | Запись в [реестр](external-sources-registry.md) — **ВСЕГДА** | — |
| Из источника извлечён полезный вывод для текущей фазы | Инсайт-файл в [external-insights/](external-insights/) + ссылка из реестра | — |
| Источник не полезен сейчас, но содержит идеи для будущих фаз или смежных тем | Реестр + теги `future-phase: X`, `topic: Y`, при необходимости `use-case: Z`; извлечённый вывод сохраняется в БЗ как задел | **НЕ** отбрасываем результат анализа |
| Источник с явным хайпом/водой: нет кода для проверки, нет метрик результата, только маркетинговые обещания | Только реестр со `stage: rejected` и пометкой «отклонено» | **НЕ** создаём инсайт в `external-insights/` |

> ⛔ **Anti-Inflation.** В базу знаний (`external-insights/`) попадает только
> польза: текущая или будущая. Проанализированный источник не отбраковывается
> полностью, если в нём есть идеи для будущих фаз; такие идеи маркируются тегами
> будущей фазы/темы и сохраняются. Полностью отклоняются только источники с
> явными признаками хайпа или воды.

### Чистота записей

- Факт изучения внешнего источника **НЕ** пишется в `CHANGELOG.md` проектов:
  changelog фиксирует governance-изменения контрактов, а не события чтения статей.
  Изучение источника живёт в реестре и (если полезно) в инсайте.

### Дополнительные правила

- **Множественные источники.** Если в одной задаче изучено несколько источников —
  каждый получает **отдельную** строку реестра и (если полезен) отдельный инсайт.
- **Повторное изучение.** Если источник уже в реестре — не дублируем строку, а
  **обновляем** существующую запись (`stage`, ссылку на инсайт, `updated`).
  Повторный анализ инициирует только Пользователь.
- **Устаревшие источники.** Если источник потерял актуальность — помечаем запись
  статусом «устарел» (deprecated) в реестре, не удаляя её: история остаётся
  трассируемой.
- **Авторство и лицензии.** Автор/организация, дата и язык фиксируются **всегда**;
  лицензия — когда применима (Traceability НФТ).

## Политики направления

- **Anti-Inflation.** Реестр и инсайты создаются по факту ценности, без
  CI-скраперов и автоматического сбора. Метаданные — только те, что нужны для
  фильтрации и синхронизации.
- **Минимальные метаданные.** Полный набор полей описан в
  [external-sources-registry.md](external-sources-registry.md); расширять состав
  полей можно только при доказанной операционной потребности.
- **Цитирование.** Каждый инсайт ссылается на запись реестра и на первоисточник
  (см. правила цитирования в
  [standards/research-profile.md](../../standards/research-profile.md)).
- **kebab-case** для всех файлов и подкаталогов.
