# File Naming

Версия: 1.0

Дата: 2026-05-26

Стандарт фиксирует единые паттерны именования файлов и каталогов для активных
артефактов репозитория. Цель — сохранить консистентную навигацию, стабильные
ссылки и предсказуемую машинную обработку документов.

## Паттерны

| Тип артефакта | Паттерн | Пример | Где применяется |
| --- | --- | --- | --- |
| Исследование | `YYYY-MM-topic.md` | `2026-05-issue-templates.md` | `research/{domain}/` |
| Стандарт | `STANDARD_NAME.md` | `FILE_NAMING.md` | `standards/` |
| Эксперимент | `exp-{slug}/` (папка) | `exp-tz-corpus/` | `research/{domain}/` |
| Профиль проекта | `{ТИП}_PROFILE.md` | `PRODUCT_PROFILE.md` | `standards/` |
| Курс | `{course-slug}/` (папка) | `prompt-engineering-ba/` | `education/` |

## Анти-паттерны

❌ `file (1).md` — дубли с номером

❌ `Исследование по задачам_финал_2.md` — кириллица + пробелы + версии в имени

❌ `2026-05-исследование--тема--контекст--ещё-длиннее.md` — перегруженное имя (контекст — во frontmatter)

## Как использовать

1. Определи тип артефакта (исследование / стандарт / эксперимент)
2. Выбери паттерн из таблицы
3. Примени к имени файла
4. Если типа нет в таблице — предложи добавить через Issue

## Правило для ИИ-ассистентов

В режиме `structured` строго следуй паттернам из таблицы. Не добавляй новые
типы артефактов или паттерны без явного указания в issue, PR comment или
другом human-authored governance документе. Если тип артефакта не покрыт
таблицей, задай вопрос или предложи отдельное issue на расширение стандарта.
