# Добавление шаблона в Хаб

> 📚 **Контекст:** этот гайд раскрывает возможность «накопление и
> переиспользование знаний» из [Product Concept](../docs/product-concept.md) и
> поддерживает ценность из [Product Vision](../docs/vision.md) — однажды добытое
> знание служит снова.

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для человека, который придумал полезный шаблон или приём и хочет внести его в
общую библиотеку Хаба, чтобы им пользовались другие команды.

## Сценарий

В работе родился удачный шаблон — структура документа, набор шагов, заготовка.
Он повторно полезен, и его стоит сделать общим достоянием через Хаб.

## Пошаговая инструкция

1. **Проверьте пользу.** Шаблон стоит добавлять, если он снимает реальную
   повторяющуюся боль, а не просто «может пригодиться» (принцип бережного роста
   библиотеки).
2. **Найдите место.** Определите подходящий раздел Хаба для шаблона и
   убедитесь, что похожего ещё нет.
3. **Оформите по правилам.** Используйте понятное имя файла в нижнем регистре с
   дефисами и добавьте служебный заголовок со статусом `draft`.
4. **Опишите назначение.** В начале шаблона коротко поясните, для чего он и
   когда применяется.
5. **Предложите изменение.** Оформите вклад как предложение (Pull Request) с
   понятным описанием пользы.
6. **Пройдите ревью.** Дождитесь обратной связи и утверждения; финальное
   решение остаётся за фаундером.

## Примеры

**Подготовить вклад в отдельной ветке:**

```bash
git switch -c add-template-my-checklist
# добавьте файл шаблона в нужный раздел
git add .
git commit -m "docs: add reusable checklist template (draft)"
git push origin add-template-my-checklist
```

**Минимальный служебный заголовок шаблона:**

```yaml
---
status: draft
version: 0.1
updated: 2026-06-09
ai-generated: false
---
```

**Чек-лист готовности шаблона к предложению:**

- [ ] Шаблон снимает реальную повторяющуюся боль.
- [ ] Имя файла в нижнем регистре с дефисами.
- [ ] Есть служебный заголовок со статусом `draft`.
- [ ] В начале объяснено назначение и когда применять.
- [ ] Похожего шаблона ещё нет.

## Что делать, если...

- **Не уверены, нужен ли шаблон** → обсудите идею с командой до оформления;
  библиотека растёт только под реальную боль.
- **Похожий шаблон уже есть** → предложите улучшение существующего вместо
  нового файла.
- **Имя файла не проходит проверку** → используйте нижний регистр и дефисы,
  без заглавных букв и пробелов.
- **Не знаете, в какой раздел положить** → задайте вопрос в обсуждении
  предложения.
