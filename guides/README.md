# Библиотека гайдов

> 📚 **Контекст:** эти гайды — практическая опора к [Product Vision](../docs/vision.md)
> и [Product Concept](../docs/product-concept.md). Vision отвечает на вопрос
> «зачем», Concept — «что и для кого», а гайды — «как это сделать».

> 📌 **Статус:** Draft — черновик для согласования фаундером единым пакетом.

## Для кого этот документ

Для **команды пользователей Хаба** — людей, а не ИИ-агентов:

- основателей и продуктовых лидеров;
- партнёров и приглашённых экспертов;
- участников команд;
- исследователей и преподавателей.

Гайды написаны человеческим языком и говорят на языке задач, а не реализации.

## Сценарий

Документ нужен, когда требуется быстро найти пошаговую инструкцию для типовой
задачи: с чего начать, как запустить проект, как взаимодействовать с
искусственным интеллектом, что делать при проблеме.

## Рекомендуемый маршрут для новичка

1. [Быстрый старт](quick-start.md) — понять, как устроена работа.
2. [Инициализация проекта](init-spoke-repo.md) — запустить первый проект.
3. [Взаимодействие с ИИ](interact-with-ai.md) — получить первый результат.
4. [Развёртывание проекта](deploy-project.md) — показать результат людям.

Когда проект закрепился в экосистеме —
[Синхронизация с проектами](sync-with-projects.md): отразите проект и его связи
в [экосистемной карте](../docs/ecosystem-map.md).

## Как читать гайды

Каждый гайд построен по единой структуре: для кого, сценарий, пошаговая
инструкция, примеры, «что делать, если...», связанные документы. Можно читать
подряд или открывать нужный гайд точечно.

## Что делать, если...

- **Не нашли нужный гайд** → начните с [Быстрого старта](quick-start.md) и
  [Решения проблем](troubleshooting.md).
- **Встретили незнакомый термин** → откройте [Глоссарий](glossary.md).
- **Нужен ответ на вопрос «зачем»** → вернитесь к [Product Vision](../docs/vision.md).
