# Решение проблем

> 📚 **Контекст:** этот гайд поддерживает ценность доверия из
> [Product Vision](../docs/vision.md) — каждый шаг понятен и обратим — и
> сопровождает все возможности из [Product Concept](../docs/product-concept.md).

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для любого пользователя Хаба, у которого что-то пошло не так и нужен спокойный
чек-лист действий.

## Сценарий

В работе возникла заминка: не запускается старт проекта, синхронизация принесла
лишнее, сайт не публикуется, результат искусственного интеллекта не тот. Этот
гайд помогает быстро найти выход.

## Пошаговая инструкция

1. **Опишите проблему одним предложением.** Что именно не получается и на каком
   шаге.
2. **Найдите профильный гайд.** Сверьтесь с таблицей ниже — у большинства задач
   есть свой гайд с разделом «Что делать, если...».
3. **Проверьте базовые условия.** Свежая копия, нужные инструменты установлены,
   связь с Хабом указана.
4. **Сделайте шаг назад.** Если изменение всё усложнило — откатитесь к
   предыдущему состоянию.
5. **Зафиксируйте и спросите.** Если проблема осталась, опишите её и обратитесь
   к команде; финальное решение остаётся за человеком.

## Примеры

**Карта проблем и профильных гайдов:**

| Симптом | Куда смотреть |
| --- | --- |

| Синхронизация принесла лишние изменения | [Откат синхронизации](rollback-sync.md) |

| Незнакомый термин | [Глоссарий](glossary.md) |
| Непонятно, с чего начать | [Быстрый старт](quick-start.md) |

**Базовые проверки перед обращением за помощью:**

```bash
# Свежая ли копия проекта
git status

# Установлены ли инструменты сборки сайта
mkdocs --version
```

**Чек-лист «прежде чем просить помощь»:**

- [ ] Проблема описана одним предложением.
- [ ] Просмотрен профильный гайд и его раздел «Что делать, если...».
- [ ] Проверены базовые условия.
- [ ] При необходимости сделан откат к рабочему состоянию.

## Что делать, если...

- **Профильного гайда нет** → начните с [Быстрого старта](quick-start.md).
- **Проблема повторяется** → зафиксируйте шаги воспроизведения, это ускорит
  помощь команды.
- **Изменение всё сломало** → вернитесь к рабочему состоянию через
  [Откат синхронизации](rollback-sync.md).
- **Ничего не помогает** → опишите проблему и обратитесь к команде за помощью.
