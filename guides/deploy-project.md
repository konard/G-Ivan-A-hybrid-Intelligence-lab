# Развёртывание проекта

> 📚 **Контекст:** этот гайд раскрывает возможность «понятная публикация
> результата» из [Product Concept](../docs/product-concept.md) и поддерживает
> ценность из [Product Vision](../docs/vision.md) — уверенно делиться
> результатом.

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для человека, который готовит результат к публикации наружу: основателя,
продуктового лидера, преподавателя, участника команды.

## Сценарий

Результат готов и проверен. Нужно опубликовать его в виде аккуратного,
читаемого и адаптивного сайта, удобного и на компьютере, и на телефоне.

## Пошаговая инструкция

1. **Подготовьте материал.** Убедитесь, что текст вычитан, ссылки работают, а
   таблицы и списки оформлены аккуратно.
2. **Разместите материал в нужном разделе.** Гайды попадают в библиотеку
   гайдов, исследования — в раздел исследований.
3. **Проверьте оформление локально.** Соберите сайт на своей машине и
   просмотрите, как выглядят страницы, таблицы и навигация.
4. **Опубликуйте изменения.** Отправьте изменения в основную ветку — публикация
   произойдёт автоматически.
5. **Проверьте результат в браузере.** Откройте опубликованный сайт и
   убедитесь, что всё читается и на компьютере, и на телефоне.

## Примеры

**Локальная проверка оформления перед публикацией:**

```bash
# Установить инструменты сборки сайта (однократно)
pip install mkdocs-material

# Собрать и просмотреть сайт локально
mkdocs serve
# Откройте http://127.0.0.1:8000 в браузере
```

**Собрать готовую версию сайта в строгом режиме:**

```bash
mkdocs build --strict
```

**Чек-лист готовности к публикации:**

- [ ] Текст вычитан, опечаток нет.
- [ ] Все ссылки открываются.
- [ ] Таблицы читаются и переносят длинные строки.
- [ ] Страница хорошо выглядит на телефоне.
- [ ] Навигация ведёт туда, куда ожидается.

## Что делать, если...

- **Таблица вылезает за край экрана** → проверьте, что материал размещён в
  поддерживаемом разделе; оформление таблиц настроено на адаптивность и
  переносы строк.
- **Страница не появилась на сайте** → убедитесь, что она добавлена в навигацию
  и изменения отправлены в основную ветку.
- **Сборка завершилась с ошибкой** → прочитайте сообщение об ошибке и проверьте
  битые ссылки; подробности в [Решении проблем](troubleshooting.md).
- **Сайт выглядит не так, как ожидалось** → сравните с локальной сборкой через
  `mkdocs serve`.
