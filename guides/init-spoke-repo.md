# Инициализация проекта

> 📚 **Контекст:** этот гайд раскрывает возможность «быстрый старт проекта» из
> [Product Concept](../docs/product-concept.md) и служит ценности из
> [Product Vision](../docs/vision.md) — превращать идею в результат без долгой
> подготовки.

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для человека, который запускает новое рабочее пространство проекта: основателя,
участника команды или партнёра, начинающего отдельное направление.

## Сценарий

Появилась идея нового проекта. Нужно создать для неё отдельное рабочее
пространство, которое с первого дня живёт по общим правилам и связано с
библиотекой знаний Хаба.

## Пошаговая инструкция

1. **Подготовьте идею.** Сформулируйте одно-двухстрочное описание проекта и
   рабочее имя.
2. **Получите шаблон старта.** Возьмите готовый «ДНК-шаблон» проекта из Хаба —
   каталог `templates/spoke/`. Это основа, из которой рождается новое рабочее
   пространство.
3. **Запустите инициализацию.** В шаблоне есть скрипт `init.sh`, который
   подставляет имя проекта, ссылку на Хаб и текущую дату во все нужные файлы.
4. **Проверьте результат.** Убедитесь, что имя проекта и описание заполнены, а
   связь с Хабом указана корректно.
5. **Сделайте первый шаг работы.** Зафиксируйте первую задачу проекта и
   переходите к [взаимодействию с ИИ](interact-with-ai.md).

## Примеры

**Интерактивный запуск (скрипт спросит значения):**

```bash
cd templates/spoke
./init.sh
```

**Неинтерактивный запуск (удобно для автоматизации):**

```bash
./init.sh \
  --name "Мой проект" \
  --description "Краткое описание проекта в одну строку" \
  --hub-url "https://github.com/G-Ivan-A/hybrid-Intelligence-lab" \
  --yes
```

**Что заполняет инициализация:**

| Что подставляется | Пример значения |
| --- | --- |
| Имя проекта | `Мой проект` |
| Описание | `Краткое описание проекта в одну строку` |
| Ссылка на Хаб | `https://github.com/G-Ivan-A/hybrid-Intelligence-lab` |
| Дата старта | сегодняшняя дата в формате `ГГГГ-ММ-ДД` |

## Что делать, если...

- **Скрипт спрашивает значения, а вы хотите без вопросов** → добавьте флаг
  `--yes` и передайте значения флагами `--name`, `--description`, `--hub-url`.
- **В файлах остались метки вида `{{...}}`** → запустите инициализацию ещё раз
  и проверьте, что указали имя проекта (оно обязательно).
- **Нужно обновить правила после старта** → перейдите к гайду
  [Синхронизация с Хабом](sync-from-hub.md).
- **Что-то пошло не так** → откройте [Решение проблем](troubleshooting.md).
