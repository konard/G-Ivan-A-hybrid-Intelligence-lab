# Быстрый старт

> 📚 **Контекст:** этот гайд — первый практический шаг к идеям из
> [Product Vision](../docs/vision.md) и [Product Concept](../docs/product-concept.md).

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для любого нового человека, который впервые открыл Хаб: основателя, партнёра,
участника команды или исследователя. Технический опыт не требуется — нужны лишь
браузер и базовое знакомство с Git.

## Сценарий

Вы получили ссылку на Хаб и хотите за 10–15 минут понять, как здесь устроена
работа, и сделать первый полезный шаг.

## Пошаговая инструкция

1. **Поймите «зачем».** Прочитайте [Product Vision](../docs/vision.md) — одну
   страницу о том, ради чего существует Хаб.
2. **Поймите «что и для кого».** Просмотрите [Product Concept](../docs/product-concept.md):
   персоны, ключевые возможности, фазы развития.
3. **Сориентируйтесь в библиотеке гайдов.** Откройте
   [оглавление гайдов](README.md) и найдите карту: какой гайд для какой задачи.
4. **Выберите первую задачу.** Чаще всего это запуск нового проекта —
   переходите к гайду [Инициализация проекта](init-spoke-repo.md).
5. **Сверьтесь со словарём.** Если встретили незнакомый термин, держите рядом
   [Глоссарий](glossary.md).

## Примеры

**Пример маршрута «первая победа» за один вечер:**

```text
Vision (5 мин) → Concept, по диагонали (10 мин)
      → Инициализация проекта (15 мин)
      → Первый запрос к ИИ (15 мин)
      → Публикация результата (10 мин)
```

**Пример первого вопроса самому себе перед стартом:**

> «Какую одну идею я хочу довести до первого работающего результата на этой
> неделе?»

Запишите ответ — он станет целью вашего первого проекта.

## Что делать, если...

- **Непонятно, с чего начать** → идите строго по шагам 1–4 выше.
- **Кажется, что слишком много документов** → читать всё сразу не нужно;
  достаточно Vision и карты гайдов, остальное открывайте по мере задач.
- **Встретили незнакомое слово** → откройте [Глоссарий](glossary.md).
- **Что-то не работает** → загляните в [Решение проблем](troubleshooting.md).
