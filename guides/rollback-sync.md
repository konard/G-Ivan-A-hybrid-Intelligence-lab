# Откат синхронизации

> 📚 **Контекст:** этот гайд поддерживает ценность доверия из
> [Product Vision](../docs/vision.md) — каждый шаг воспроизводим и обратим — и
> дополняет возможность «накопление и переиспользование знаний» из
> [Product Concept](../docs/product-concept.md).

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для участника проекта, который провёл синхронизацию с Хабом и хочет вернуть
проект к состоянию до неё.

## Сценарий

После [синхронизации с Хабом](sync-from-hub.md) в проект попали изменения,
которые сейчас не нужны или конфликтуют с локальной работой. Нужно безопасно
откатиться к предыдущему состоянию.

## Пошаговая инструкция

1. **Не паникуйте.** Каждый шаг обратим, если перед синхронизацией была сделана
   контрольная точка.
2. **Найдите точку возврата.** Определите состояние проекта до синхронизации.
3. **Просмотрите, что откатывается.** Сравните текущее состояние с точкой
   возврата, чтобы понимать последствия.
4. **Выполните откат.** Верните проект к выбранному состоянию.
5. **Проверьте результат.** Убедитесь, что нужная локальная работа сохранена, а
   нежелательные изменения убраны.

## Примеры

**Посмотреть историю и найти точку до синхронизации:**

```bash
git log --oneline -n 10
```

**Откатить незавершённую синхронизацию в рабочей ветке:**

```bash
# Отменить непринятые изменения в рабочих файлах
git restore .

# Удалить временную ветку синхронизации
git switch main
git branch -D sync-from-hub
```

**Вернуться к конкретной контрольной точке (осторожно):**

```bash
# Посмотреть отличия перед откатом
git diff <commit-точки-возврата> HEAD

# Вернуть состояние к выбранной точке
git reset --hard <commit-точки-возврата>
```

**Чек-лист после отката:**

- [ ] Нежелательные изменения убраны.
- [ ] Нужная локальная работа сохранена.
- [ ] Проект собирается и проходит локальные проверки.
- [ ] Команда уведомлена об откате, если синхронизация была общей.

## Что делать, если...

- **Контрольной точки не было** → восстановите нужные файлы по истории
  изменений; на будущее всегда делайте точку перед синхронизацией.
- **Откат затронул нужную работу** → восстановите отдельные файлы из истории, а
  не весь проект целиком.
- **Не уверены, к какой точке возвращаться** → сравните состояния через `git
  diff` и выберите осознанно.
- **Ситуация запуталась** → откройте [Решение проблем](troubleshooting.md).
