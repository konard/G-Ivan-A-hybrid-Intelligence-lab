# Синхронизация с проектами

> 📚 **Контекст:** этот гайд поддерживает [Product Vision](../docs/vision.md) и
> [Product Concept](../docs/product-concept.md) и опирается на
> [Экосистемную карту Хаба](../docs/ecosystem-map.md). Vision отвечает «зачем»,
> Concept — «что и для кого», карта — «как Хаб связан с другими проектами», а
> этот гайд — «как поддерживать карту и связи проекта в актуальном виде».

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для **команды пользователей Хаба** — людей, а не ИИ-агентов:

- фаундера и продуктовых лидеров;
- партнёров и приглашённых экспертов;
- участников команд проектов.

Гайд нужен тем, кто отвечает за то, чтобы их проект был правильно отражён в
[Экосистемной карте Хаба](../docs/ecosystem-map.md) и чтобы связи проекта с
другими проектами оставались актуальными.

## Сценарий

Хаб хранит **полный граф экосистемы** в [ecosystem-map.md](../docs/ecosystem-map.md):
какие проекты входят в экосистему и как они связаны. Каждый проект по принципу
**«Need-to-Know»** хранит только **частичный граф** — те проекты, с которыми
связан по функционалу (`.hub-ecosystem.json` для инструментов и
`docs/ecosystem-context.md` для людей).

Синхронизация с проектами — это процесс, который держит обе стороны
согласованными:

- когда меняется экосистема (новый проект, новая связь, смена статуса) —
  обновляется полный граф в Хабе;
- когда меняется полный граф — частичный граф проекта обновляется
  **полуавтоматически**: скрипт Smart Sync предлагает изменения, а человек их
  подтверждает.

Этот гайд описывает, как пройти процесс, ничего не сломав и сохранив контроль
за человеком.

## Пошаговая инструкция

1. **Откройте карту экосистемы.** Зайдите в
   [ecosystem-map.md](../docs/ecosystem-map.md) и посмотрите текущий полный граф:
   матрицу проектов и граф связей.
2. **Найдите свой проект.** Проверьте строку проекта в матрице: статус, тип и
   связь с Хабом. Если проекта ещё нет — вы будете его добавлять (см.
   [Примеры](#примеры)).
3. **Проверьте связи.** Сверьте функциональные связи проекта в графе с тем, как
   проект работает на самом деле: появились новые зависимости от других проектов
   или какие-то связи отпали?
4. **Внесите изменения через PR в Хаб.** Обновите матрицу и граф в
   [ecosystem-map.md](../docs/ecosystem-map.md) и откройте Pull Request. Полный
   граф — единый источник истины, поэтому правки идут в Хаб, а не в копии.
5. **Запустите Smart Sync.** После принятия PR запустите синхронизацию на стороне
   проекта — скрипт прочитает полный граф и **предложит** обновление частичного
   графа проекта (он ничего не меняет молча).
6. **Подтвердите частичный граф.** Просмотрите предложенные изменения и осознанно
   подтвердите их: обновите `.hub-ecosystem.json` (для инструментов) и
   `docs/ecosystem-context.md` (для людей проекта). Решение остаётся за человеком.

## Примеры

**Добавить новый проект в экосистему:**

```text
1. Создать репозиторий проекта по шаблону (templates/htom/ или templates/spoke/).
2. Инициализировать профиль проекта .hub-profile.json (тип, стек, фаза).
3. Добавить строку в матрицу и узел в граф в docs/ecosystem-map.md → PR в Хаб.
4. Запустить Smart Sync → подтвердить .hub-ecosystem.json нового проекта.
```

**Изменить связь существующего проекта:**

```text
1. В docs/ecosystem-map.md обновить ребро графа (добавить/убрать связь) → PR в Хаб.
2. После принятия PR запустить Smart Sync в затронутых проектах.
3. Подтвердить обновлённый частичный граф (.hub-ecosystem.json,
   docs/ecosystem-context.md) в каждом затронутом проекте.
```

**Удалить проект из экосистемы:**

```text
1. В docs/ecosystem-map.md убрать строку из матрицы и узел из графа,
   а также все связи с этим проектом → PR в Хаб.
2. Запустить Smart Sync в проектах, которые были с ним связаны.
3. Подтвердить, что связь удалена из их частичного графа.
```

**Чек-лист перед синхронизацией:**

- [ ] Полный граф в [ecosystem-map.md](../docs/ecosystem-map.md) обновлён через PR.
- [ ] Smart Sync запущен и показал предлагаемые изменения.
- [ ] Изменения частичного графа просмотрены человеком.
- [ ] `.hub-ecosystem.json` и `docs/ecosystem-context.md` подтверждены осознанно.

## Что делать, если...

- **Не нашли свой проект в карте** → добавьте его (см.
  [Примеры](#примеры) и раздел «Добавление нового проекта» в
  [ecosystem-map.md](../docs/ecosystem-map.md#добавление-нового-проекта)).
- **Связи в карте не совпадают с реальностью** → обновите граф в
  [ecosystem-map.md](../docs/ecosystem-map.md) через PR, затем синхронизируйте
  проекты.
- **Smart Sync предлагает лишние изменения** → ничего не подтверждайте, обсудите
  расхождение с командой; скрипт только предлагает, решение за человеком.
- **Нужно обновить правила и шаблоны, а не связи** → это другой процесс,
  см. [Синхронизация с Хабом](sync-from-hub.md).
- **Непонятен термин или принцип** → откройте [Глоссарий](glossary.md) и раздел
  [«Need-to-Know»](../docs/ecosystem-map.md#принцип-need-to-know) в карте.
