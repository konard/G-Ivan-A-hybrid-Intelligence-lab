# Синхронизация с Хабом

> 📚 **Контекст:** этот гайд поддерживает ценность из
> [Product Vision](../docs/vision.md) — знания служат снова и снова — и
> возможность «накопление и переиспользование знаний» из
> [Product Concept](../docs/product-concept.md).

> 📌 **Статус:** Draft — черновик для согласования.

## Для кого этот документ

Для участника проекта, который хочет, чтобы его рабочее пространство опиралось
на свежие правила, стандарты и шаблоны из общей библиотеки Хаба.

## Сценарий

Хаб — источник истины для общих правил и шаблонов. Со временем в нём появляются
обновления. Этот гайд описывает, как подтянуть актуальные материалы в свой
проект, ничего не сломав.

## Пошаговая инструкция

1. **Сверьтесь с источником истины.** Откройте Хаб и посмотрите, что
   изменилось: новые стандарты, обновлённые шаблоны, правки правил.
2. **Сделайте контрольную точку.** Зафиксируйте текущее состояние проекта,
   чтобы при необходимости вернуться назад (см.
   [Откат синхронизации](rollback-sync.md)).
3. **Получите обновления.** Подтяните свежие материалы из Хаба в проект.
4. **Просмотрите изменения.** Проверьте, что именно поменялось, и оцените, всё
   ли подходит вашему проекту.
5. **Примите изменения осознанно.** Сохраните то, что нужно проекту; спорные
   места обсудите с командой до принятия.

## Примеры

**Сравнить состояние своего проекта со свежим Хабом:**

```bash
# Однократно: указать Хаб как дополнительный источник
git remote add hub https://github.com/G-Ivan-A/hybrid-Intelligence-lab.git

# Получить свежие данные Хаба
git fetch hub

# Посмотреть, какие правила/шаблоны изменились
git diff HEAD hub/main -- templates/ standards/ governance/
```

**Подтянуть обновления и просмотреть их перед принятием:**

```bash
# Создать рабочую ветку под синхронизацию
git switch -c sync-from-hub

# Внести обновления и просмотреть изменения построчно
git status
git diff
```

**Чек-лист перед принятием синхронизации:**

- [ ] Сделана контрольная точка для возможного отката.
- [ ] Просмотрены все изменения.
- [ ] Локальные правки проекта не затёрты.
- [ ] Спорные места согласованы с командой.

## Что делать, если...

- **Изменений слишком много** → синхронизируйте по частям: сначала правила,
  затем шаблоны.
- **Обновление конфликтует с локальными правками** → сохраните свою версию,
  обсудите расхождение с командой, примите решение осознанно.
- **Синхронизация принесла лишнее** → перейдите к
  [Откату синхронизации](rollback-sync.md).
- **Непонятно, что поменялось** → откройте журнал изменений Хаба и
  [Глоссарий](glossary.md).

## Автоматизация (Smart Sync)

Вместо ручного `git diff` можно использовать клиент Smart Sync
`tools/sync-from-hub.sh`. Он читает реестр шаблонов Хаба
(`templates/manifest.json`) и локальный профиль проекта `.hub-profile.json`,
фильтрует шаблоны по релевантности (тип проекта, стек, фаза) и применяет
обновления безопасно — не затирая локальные правки.

```bash
# Однократно: создать профиль проекта (тип, стек, фаза)
tools/sync-from-hub.sh --init

# Посмотреть отчёт: что новое и что обновилось (без изменений)
tools/sync-from-hub.sh --report

# Применить безопасно: рядом с локальными файлами создаются <имя>.hub-new.*
tools/sync-from-hub.sh --apply

# Перезаписать локальные файлы версиями из Хаба (осознанно!)
tools/sync-from-hub.sh --apply --force
```

В безопасном режиме (без `--force`) обновление сохраняется как `*.hub-new.md`
рядом с вашим файлом — сравните и смержите вручную. Если что-то пошло не так,
см. [Откат синхронизации](rollback-sync.md).
